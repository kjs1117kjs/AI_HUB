013분 요약

아래 순서대로만 따라 하면 하나의 화면이 완성됩니다.

① 조회조건→ ② 컬럼→ ③ 데이터→ ④ 옵션 · 버튼→ ⑤ 미리보기→ ⑥ 저장 · 내보내기
  1. 조회조건 추가 — 조건명을 입력하고 📅 일자 🔤 텍스트 같은 유형 버튼을 누릅니다.
  2. 컬럼 추가 — 컬럼명을 쉼표로 나열하면 여러 개가 한 번에 만들어집니다. (예: 매장코드, 매장명, 매출액)
  3. 데이터 채우기 — 🎲 샘플로 자동 생성하거나 📋 붙여넣기로 엑셀 표를 그대로 붙여 넣습니다.
  4. 옵션 지정 — 대상 패널의 체크박스열 · 행번호 · 합계행 · 페이지, 그리고 패널 버튼을 정합니다.
  5. 미리보기 — 👁 미리보기 모드로 실제 사용자가 보는 화면을 확인합니다.
  6. 보관 — 💾 저장(JSON)으로 구성을 남기고, 🌐 HTML 내보내기로 공유용 화면을 뽑습니다.
가장 빠른 시작 무엇부터 할지 모르겠다면 📚 예시 화면에서 비슷한 화면을 불러온 뒤 이름과 컬럼만 고치는 방법을 권합니다.

03조회조건 만들기

화면 상단의 검색 영역을 구성합니다. 조건은 왼쪽부터 2개씩 한 줄에 자동 배치됩니다.

기본 사용법

  1. 조회조건명에 라벨을 입력합니다. (예: 조회일자)
  2. 필요하면 옵션을 먼저 체크합니다 — 기간(~) 두 칸 범위 입력, 필수(*) 라벨에 빨간 별, 확장 확장조회로 숨김
  3. 유형 버튼을 누르면 조건이 추가됩니다.

유형

버튼설명
📅 일자달력 선택 입력칸. 기간 체크 시 시작 ~ 종료
🗓️ 월연월 선택기
📆 년도 🕒 분기연도 · 분기 콤보
🔤 텍스트자유 입력
🔢 숫자숫자 입력. 기간 체크 시 최소~최대

🏬 매장 📦 상품내장된 매장 · 상품 검색 팝업
▫️ 빈칸줄 맞춤용 빈 칸. 조건 1개만 있는 줄을 만들 때 사용

04조회 · 엑셀 버튼

검색 영역 오른쪽 버튼과 그리드 위 엑셀 버튼을 직접 정의합니다. 기본값은 조회 와 엑셀다운로드 하나씩입니다.

  1. 조회/엑셀 버튼 섹션에 버튼명을 입력합니다. 쉼표로 여러 개를 한 번에 넣을 수 있습니다. (예: 확장조회,초기화)
  2. 강조 체크 시 파란색 버튼으로 만들어집니다.
  3. ＋ 조회영역 또는 ＋ 엑셀영역을 누릅니다.
  4. 지우려면 편집 모드에서 버튼 안의 ×를, 처음 상태로 돌리려면 기본값을 누릅니다.
버튼명동작
이름에 ‘확장’ 포함확장조건 영역 열기/닫기 (미리보기 모드)
그 외 조회영역 버튼조회 실행 — 결과 그리드와 건수를 갱신
엑셀영역 버튼대상 패널을 .xlsx 파일로 내려받기 (그룹헤더 · 합계행 포함)

05레이아웃과 패널

결과 영역은 1~3단으로 나눌 수 있고, 각 단에 여러 개의 그리드(패널)를 쌓을 수 있습니다. 마스터-디테일 화면이 여기서 만들어집니다.

패널 추가

  1. 레이아웃 · 패널에서 컬럼수 1단 2단 3단을 고릅니다.
  2. 패널명을 입력하고, 놓을 단을 선택한 뒤 ＋ 패널 추가를 누릅니다.
  3. 같은 단에 패널을 두 개 추가하면 위아래로 쌓입니다.

대상 패널

중요 컬럼 추가 · 데이터 입력 · 패널 옵션은 모두 ‘대상 패널’에 선택된 패널에 적용됩니다. 여러 패널을 쓸 때는 작업 전에 대상 패널을 먼저 바꿔 주세요.
체크박스열

행번호
No 열 표시

06컬럼 추가

결과 그리드의 열을 정의합니다. 한 번에 여러 개를 만들 수 있습니다.

입력 항목설명
컬럼명쉼표로 나열하면 여러 컬럼이 한 번에 추가됩니다. 예: 매장코드, 매장명, 매출액
그룹명입력하면 2줄 헤더가 됩니다. 같은 그룹명을 연달아 지정한 컬럼끼리 자동 병합됩니다.

합계행이 잡히지 않는다면 숫자로 인식되어야 합계가 계산됩니다. 유형을 숫자로 하거나 정렬을 오른쪽, 표시형식을 1,000·₩·% 중 하나로 지정하세요.

08조회조건 팝업 (🪟)

조회조건 칸 옆의 “선택” 상자를 눌렀을 때 뜨는 코드·명칭 검색 팝업입니다.

  1. 팝업 구성 → 🪟 팝업 만들기에서 팝업명 · 팝업 안의 조회조건 · 결과 컬럼 · 예시 데이터를 정의합니다. 또는 📚 팝업예시에서 매장선택 · 상품선택 · 사원선택 등 9종을 바로 생성합니다.
  2. 만들어진 칩에서 ＋를 누르면 현재 화면의 조회조건으로 추가됩니다.
  3. 칩의 👁 미리보기, ✎ 수정, ✕ 삭제로 관리합니다.

미리보기 모드에서 그 조건의 선택 상자를 클릭하면 실제 팝업이 열리고, 행을 고르면 조건 값으로 채워집니다.

09정보 · 작업 팝업 (🗂️) — 링크 클릭 시 열기

결과 그리드의 코드를 클릭했을 때 뜨는 상세 정보 화면 또는 처리용 입력 폼입니다. 연결하려면 두 가지가 모두 맞아야 합니다 — ① 팝업이 만들어져 있을 것, ② 컬럼 유형이 링크(코드)일 것.

1단계 · 팝업 만들기

정보 팝업 섹션에서 🗂️ 정보팝업 만들기로 직접 구성하거나, 📚 팝업예시에서 18종을 바로 생성합니다.

종류예시특징
상세형상품정보 · 매장정보 · 회원정보 · 주문상세 · 쿠폰정보여러 섹션(및 탭)으로 나뉜 상세 조회 · 저장/닫기
작업형비밀번호 변경 · 로그인 잠금해제 · 포인트 조정 · 재고 조정 · 알림 재발송 · 단말 인증 초기화좁은 폼 + 안내문 + 전용 처리 버튼
그리드형신용카드 결제내역 · 주문 상품내역 · 포인트 적립·사용 내역 · 재고 입출고 이력 · 로그인 이력 · 정산 상세내역 · 쿠폰 사용내역표 자체를 담는 넓은 팝업. 합계행 · 가로 스크롤 · 요약 폼/탭/안내문과 조합

2단계 · 컬럼에 연결하기 (두 가지 방법)

방법 A — 컬럼을 만들면서 연결 (권장)

  1. 컬럼 추가에서 유형을 링크(코드)로 선택
  2. 바로 옆 (링크팝업 없음) 목록에서 연결할 팝업 선택
  3. 컬럼명 입력 후 ＋ 컬럼

방법 B — 이미 만든 컬럼에 연결
정보 팝업 칩의 🔗 버튼을 누르면 대상 패널의 링크(코드) 컬럼 전부에 그 팝업이 연결됩니다.

3단계 · 확인

👁 미리보기 모드로 전환한 뒤 링크 셀을 클릭하면 팝업이 열립니다.

주의 · 🔗는 일괄 연결입니다. 한 화면에서 사용자ID → 비밀번호 변경, 사용자명 → 로그인 잠금해제처럼 컬럼마다 다른 팝업을 붙이려면 방법 A로 하나씩 지정하세요.
· 셀 값이 비어 있는 행은 링크가 생기지 않습니다.
· 연결 전에 대상 패널이 맞는지 확인하세요.
· 편집 모드에서는 클릭해도 열리지 않습니다.
팝업 자체를 미리 보고 싶다면 칩의 👁를 누르면 컬럼 연결과 상관없이 팝업 모양만 바로 확인할 수 있습니다. ✎로 필드 구성을 수정할 수도 있습니다.

11저장 · 불러오기 · 내보내기

기능내용
💾 저장(JSON)구성 전체(조건 · 팝업 · 패널 · 컬럼 · 데이터 · 버튼 · 설명)를 .json 파일로 저장. 이어서 작업하려면 이 파일을 보관하세요. 009부터는 화면 목록의 모든 화면이 한 파일(프로젝트)로 저장되며, 이전 버전에서 저장한 단일 화면 JSON도 그대로 불러올 수 있습니다.
📂 불러오기저장한 JSON을 열어 구성을 그대로 복원
🌐 HTML 내보내기완성된 화면을 단일 .html 파일로 저장 (공유 · 첨부용). 009부터 화면ID · 작성자 · 버전 메타와 주석 핀 · 주석 목록이 함께 담기고, 010부터는 내보낸 화면에서도 탭 전환과 컬럼 너비 조절(헤더 가장자리 드래그)이 동작합니다.
📑 화면정의서(009) 조회조건 표(타입 · 기간 · 필수 · 확장 · 옵션), 버튼 구성, 패널별 정의 표(그리드: 컬럼 정의 / 폼: 필드 정의 / 요약카드: 카드 목록 / 안내문: 내용), 팝업 목록, 화면 설명, 주석 목록 · 화면 표시 문구를 정리한 인쇄용 정의서 HTML을 내려받습니다. 기획 산출물로 그대로 사용할 수 있습니다.
엑셀영역 버튼대상 패널을 .xlsx로 저장 (2줄 그룹헤더 병합 · 합계행 포함)
전체 초기화모든 구성을 처음 상태로 되돌림 (되돌리기 불가)
꼭 알아 두세요 · 작업 내용은 브라우저에 자동 저장되지 않습니다. 창을 닫기 전에 반드시 JSON으로 저장하세요.
· 내보낸 HTML은 화면 스냅샷입니다. 링크 클릭 시 팝업 열기, 확장조회 토글, 정렬 같은 동작은 담기지 않습니다. 재편집·동작 확인은 JSON + 구성기로 하세요.

12예시 불러오기

비슷한 화면을 불러와 고쳐 쓰는 방식이 가장 빠릅니다.

📚 예시 화면
조회 화면 20종 — 매출 현황 · 상품마스터 · 손익계산서 · 공지사항 · 매장/주문/회원/쿠폰/정산 관리 · 시스템명칭 · 권한관리 · 사이드메뉴 · 키오스크 · 비밀번호 임의변경 · 포인트 조정 · 재고조정 · 발송이력 · 단말 인증관리 등. 현재 구성이 대체됩니다.
📚 팝업예시
팝업 27종 — 조회조건 팝업 9 + 정보·작업·그리드 팝업 18. 현재 구성에 추가되고 미리보기로 열립니다.

팝업예시는 팝업 구성과 정보 팝업 두 섹션 어느 버튼으로 열어도 27종 모두 볼 수 있으며, 누른 쪽이 위로 정렬됩니다.

13자주 쓰는 구성 레시피

① 2줄 그룹헤더 만들기

컬럼 추가에서 그룹명에 같은 값을 넣어 컬럼을 연달아 추가하면 상단이 자동 병합됩니다.
예: 그룹명 매출현황으로 총매출, 총할인, 실매출을 한 번에 추가 → 세 컬럼 위에 매출현황 한 칸.

② 마스터-디테일(2패널) 화면

  1. 레이아웃 컬럼수 2단 선택
  2. 패널명 대표명칭(1단), 세부명칭(2단)으로 각각 추가
  3. 대상 패널을 바꿔 가며 컬럼과 데이터를 채웁니다

③ 링크 클릭 → 처리 팝업

📚 팝업예시 → 작업형 팝업 생성 → 컬럼을 링크(코드) + 해당 팝업으로 추가 → 미리보기에서 클릭. (자세히는 09장)

④ 확장조회 화면

자주 안 쓰는 조건에 확장 체크 → 조회영역 버튼에 확장조회 추가 → 미리보기에서 토글 확인.

⑤ 조회 전 빈 화면 표현

미리보기에서 조회 버튼을 누르기 전에는 건수 표시가 비어 있습니다. 조회 버튼을 눌러 결과가 채워지는 흐름 그대로 시연할 수 있습니다.

14자주 묻는 질문

컬럼을 추가했는데 아무 데도 안 보입니다.

대상 패널이 다른 패널로 선택돼 있을 가능성이 큽니다. 대상 패널을 바꿔 확인하세요.

링크를 눌러도 팝업이 안 열립니다.

① 편집 모드인지, ② 컬럼 유형이 링크(코드)인지, ③ 그 컬럼에 팝업이 연결됐는지(칩의 🔗 또는 컬럼 추가 시 링크팝업 선택), ④ 셀 값이 비어 있지 않은지 확인하세요.

합계행에 숫자가 안 나옵니다.

해당 컬럼이 숫자로 인식되어야 합니다. 유형 숫자, 정렬 오른쪽, 표시형식 1,000 중 하나 이상을 지정하세요.

확장조건이 미리보기에서 계속 보입니다.

이름에 ‘확장’이 들어간 조회영역 버튼이 없으면 접는 기능이 없다고 보고 모두 표시합니다. 확장조회 버튼을 추가하세요.

내보낸 HTML에서 팝업이 안 열립니다.

정상 동작입니다. 내보내기는 화면 모양을 담은 스냅샷이며 동작 스크립트는 포함하지 않습니다.

작업하던 화면을 다음에 이어서 하려면?

💾 저장(JSON)으로 파일을 남기고, 다음에 📂 불러오기로 여세요. 브라우저에는 저장되지 않습니다.

화면이 좁아 빌더가 잘립니다.

빌더 영역은 가로로 접히므로 브라우저 폭을 넓히거나 확대 배율을 낮추면 편합니다.

15편집 도우미 기능 (009)

009 버전에 추가된, 구성 작업을 빠르게 해 주는 기능들입니다.

기능사용 방법
드래그 정렬조건 · 컬럼의 컨트롤에 있는 ⠿ 핸들을 잡고 원하는 위치로 끌어다 놓습니다. 파란 점선이 표시된 곳으로 이동합니다. (컬럼은 같은 패널 안에서만)
실행취소 · 다시실행Ctrl+Z / Ctrl+Y 또는 ↩ 실행취소 ↪ 다시실행 버튼. 최대 80단계까지 되돌릴 수 있습니다.
조건 · 컬럼 복제컨트롤의 ⧉ 버튼. 컬럼 복제는 행 데이터 값까지 함께 복사됩니다.
패널 복제대상 패널 줄의 ⧉ 패널 복제. 컬럼 · 데이터 · 버튼 · 옵션을 그대로 가진 독립 사본을 만듭니다.

고정 컬럼대상 패널의 고정(1~3열) 선택. 가로 스크롤 시 행번호 · 체크박스열과 함께 왼쪽에 고정됩니다.
필터행대상 패널의 필터행 체크. 헤더 아래 필터 입력 행이 생기고 실제로 걸러집니다. 합계행과 건수도 필터 결과 기준으로 바뀝니다.

16다중 화면 · 화면 정보 · 주석 · 텍스트 · 정의서 (009)

화면 하나짜리 데모 도구를 넘어, 프로젝트 화면정의 도구로 쓰기 위한 기능입니다.

화면 목록 — 한 파일에서 여러 화면 관리

빌더 맨 위 화면 목록에서 ＋ 새 화면 · ⧉ 화면 복제 · 화면 삭제로 화면을 여러 개 만들고 선택 상자로 전환합니다. 전환하면 현재 작업이 자동 보관되고, 💾 저장(JSON) 한 번으로 모든 화면이 프로젝트 파일 하나에 저장됩니다.

화면 정보 — 화면ID · 작성자 · 버전

화면 목록 옆에 화면ID(예: SCR-POS-001) · 작성자 · 버전을 입력하면 JSON · HTML 내보내기 · 화면정의서 · 파일명에 함께 기록됩니다. 화면이 많아질수록 화면ID를 붙여 두는 것을 권합니다.

텍스트(📝) 모드 — 화면 위에 문구 직접 표시

  1. 📝 텍스트 모드를 눌러 켭니다. (화면 영역에 파란 점선 테두리 표시)
  2. 화면 아무 곳이나 클릭하면 그 자리에 문구가 바로 표시되고 입력 창이 열립니다. 예: ※ 마감 후에는 수정이 불가합니다
  3. 입력 창에서 글자 색(빨강 · 파랑 · 초록 · 회색 · 검정) · 크기(12~17px) · 형광펜 배경을 고르면 즉시 미리보기로 반영됩니다.
  4. 문구는 드래그로 옮기고, 클릭해서 수정 · 삭제합니다. 내용을 비우고 저장하면 자동 삭제됩니다.
주석(📌)과의 차이 주석은 번호 핀 + 별도 목록으로 정리되는 개발 전달 스펙용이고, 텍스트는 화면에 그대로 보이는 주의사항 · 안내 문구용입니다. 둘 다 HTML 내보내기와 화면정의서에 포함됩니다.

주석(📌) 모드 — 화면 위에 스펙 달기

  1. 📌 주석 모드를 눌러 켭니다. (화면 영역에 빨간 점선 테두리 표시)
  2. 화면 아무 곳이나 클릭하면 번호 핀이 찍히고 설명 입력 창이 열립니다. 예: 클릭 시 인증 초기화 팝업 호출
  3. 핀은 드래그로 옮기고, 클릭해서 수정 · 삭제합니다. 화면 아래 주석 목록에 전체가 정리됩니다.
  4. 주석 모드를 끄면 화면이 다시 정상 동작합니다. 핀은 계속 표시됩니다.
주석은 산출물에 함께 나갑니다 🌐 HTML 내보내기에는 핀과 주석 목록이, 📑 화면정의서에는 번호별 주석 표가 포함됩니다. "이 버튼 클릭 시 ○○ 처리" 같은 개발 전달 스펙을 화면 위에 직접 적어 두세요.

📑 화면정의서

조회조건 · 버튼 · 패널별 컬럼 정의 · 팝업 · 주석을 표로 정리한 인쇄용 문서를 내려받습니다. 자세한 구성은 11장 표를 참고하세요.

17패널 유형 — 폼 · 요약카드 · 안내문 · 상태 뱃지 (010)

조회조건 + 표(그리드)에 머물던 화면 구성을 넓히는 기능입니다. 패널을 추가할 때 유형을 고르면 됩니다.

패널 유형 4종

유형용도내용 추가 방법
그리드(표)조회 결과 목록 (기본)컬럼 추가 · 결과 데이터
폼(입력·상세)등록 · 수정 화면, 목록 옆 상세 정보 영역폼 필드 · 카드 섹션에서 필드 추가 (텍스트 · 콤보 · 일자 · 여러줄 · 읽기전용 · 체크박스 · 안내문구)

선택형 조건 신규 유형 (010) — 버튼그룹 · 토글 · 다중선택

유형용도동작
🔳 버튼그룹일별 · 주별 · 월별처럼 붙어 있는 버튼 중 하나 선택 (세그먼트)미리보기와 내보낸 HTML에서 클릭 시 선택이 바뀝니다
🎚️ 토글사용/미사용 같은 On · Off 스위치. 옵션을 비우면 자동으로 사용,미사용클릭 시 스위치가 전환되고 라벨(첫 번째/두 번째 옵션)이 바뀝니다
☑ 다중선택드롭다운을 열어 여러 항목을 체크로 고르는 콤보버튼 클릭으로 목록을 열고, 체크 수에 따라 "N개 선택"으로 표시됩니다

선택형 조건 섹션에도 뒤 버튼 입력칸이 있어, 콤보 · 라디오 · 체크박스 · 신규 3종 모두 조건 뒤에 버튼을 붙일 수 있습니다.

조회조건 뒤 버튼 (010)

조건 입력칸 뒤에 중복확인 · 인증요청 · 초기화 같은 버튼을 붙일 수 있습니다. 방법은 두 가지: ① 조회조건 추가 시 뒤 버튼 칸에 버튼명을 쉼표로 입력하고 조건을 추가하거나, ② 이미 있는 조건의 컨트롤에서 B＋를 눌러 버튼명을 입력합니다. 이름=색 형식으로 버튼 색을 지정할 수 있습니다 — 파랑 · 빨강 · 초록 · 주황 · 흰색, 미지정 시 기본 회색. 예: 중복확인=파랑,초기화. 편집 모드에서 버튼 위에 마우스를 올리면 나오는 ×로 제거합니다. 화면정의서의 조회조건 표에도 "뒤 버튼"으로 함께 기록됩니다.

탭마다 다른 화면 구성 (010)

화면 설정에서 탭을 적용하면, 이제 패널이 탭별로 나뉩니다. 원하는 탭을 클릭한 상태에서 ＋ 패널 추가를 누르면 그 탭에 추가되고, 다른 탭에는 보이지 않습니다. 대상 패널 목록에는 [탭명]이 붙어 어느 탭 소속인지 표시되며, 다른 탭의 패널을 선택하면 화면도 그 탭으로 자동 전환됩니다. 탭 목록을 줄이면 사라진 탭의 패널은 첫 번째 탭으로 이동합니다. HTML 내보내기 시 모든 탭의 화면이 담겨 내보낸 파일에서도 탭을 클릭해 전환할 수 있습니다.

폼 필드 다루기

필드명을 쉼표로 나열하면 여러 개가 한 번에 추가됩니다. 콤보 · 체크박스는 옵션 칸에 쉼표로 항목을, 읽기전용 · 안내문구는 같은 칸에 표시할 값을 넣습니다. 필드의 컨트롤(▲▼⧉*↔✕)로 순서 이동 · 복제 · 필수(*) · 한 줄 차지(↔) · 삭제를 조정합니다. 여러줄 · 안내문구는 자동으로 한 줄을 차지합니다.

상태 뱃지 컬럼

컬럼 유형에서 상태 뱃지를 고르면 값이 색 뱃지로 표시됩니다(미리보기 기준). 옵션에 영업중=초록,휴점=주황,폐점=빨강처럼 값=색을 지정하거나, 비워 두면 값의 키워드(정상 · 완료 → 초록, 대기 · 잠금 → 주황, 만료 · 오류 → 빨강)로 자동 판별합니다. 색은 초록 · 빨강 · 주황 · 파랑 · 회색 5종입니다.

유형별 제한 · 컬럼 추가 · 행 · 샘플 · 필터행 · 고정컬럼 · 엑셀 다운로드는 그리드 패널에서만 동작합니다.
· ＋ 필드는 폼 패널, ＋ 카드는 요약카드 패널이 대상 패널일 때만 추가됩니다. 대상 패널 목록에 [폼] · [카드] · [안내] 표시로 유형을 확인하세요.
