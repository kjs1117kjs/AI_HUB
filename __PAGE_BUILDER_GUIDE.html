01이런 도구입니다

화면 빌더는 코드를 몰라도 홈페이지·랜딩·대시보드 같은 화면을 조립할 수 있는 단일 HTML 도구입니다. 결과물은 두 가지로 뽑습니다.

  • 🌐 HTML 내보내기 — 만든 화면 그대로 열리는 독립 HTML. 한 파일로 받거나 페이지별 파일로 나눠 받을 수 있고(11장), 두 방식 모두 페이지 이동(메뉴·버튼·사이드메뉴 클릭)이 실제로 동작합니다. 설계 검토용은 물론 간단한 실사용 페이지로도 쓸 수 있습니다.
  • 📑 화면정의서 — 페이지 구조·블록 목록·주석을 표로 정리한 인쇄용 문서. 개발팀에 전달하는 설계서가 됩니다.
Note
브라우저에서 파일을 여는 것만으로 동작하며 서버·설치가 필요 없습니다. 작업 내용은 JSON 저장 파일로 보관하고 다시 불러올 수 있습니다.

02화면 구성과 두 가지 모드

  • 상단 도구막대 — 페이지 관리(추가·이름·복제·삭제·순서 이동·하위 지정) / 행 추가 / 메뉴바·푸터 스위치 / 템플릿 / 저장·불러오기 / 내보내기(한 파일·페이지별) / 주석 모드 / 텍스트 모드 / 실행취소.
  • 캔버스 — 실제 화면 모습 그대로 편집하는 영역. 텍스트는 화면에서 직접 클릭해 입력합니다(입력 후 바깥 클릭 시 저장).
  • 블록 도구 — 블록에 마우스를 올리면 우상단에 나타나는 작은 도구(속성 · ▲▼ 이동 · ⧉ 복제 · ✕ 삭제).

편집 ↔ 미리보기

우상단 👁 미리보기를 누르면 점선·도구가 모두 사라진 완성 모습을 확인할 수 있고, 메뉴·버튼 클릭으로 페이지 이동도 실제로 동작합니다. 다시 누르면 편집으로 돌아옵니다.

03가장 빠른 시작 — 예시 템플릿 60종

도구막대의 📚 예시 템플릿 (60종)… 콤보에서 원하는 유형을 고르면 화면 전체가 그 예시로 바뀝니다(실행취소 가능). 유명 홈페이지들의 대표 레이아웃을 본떠 만들었으므로, 비슷한 유형을 골라 내용만 바꾸는 것이 가장 빠른 사용법입니다.

Note — (2단) 템플릿이란 010
이름 뒤에 (2단)이 붙은 ㉛~㊺ 템플릿 15종은 상단 메뉴바가 상위 메뉴 + 하위(드롭다운) 메뉴 구조로 만들어진 견본입니다. 각 하위 항목이 실제 페이지와 연결되어 있어(홈 + 하위 4페이지, 총 5페이지 구성) 드롭다운 문법 상위(하위1/하위2)의 사용법을 그대로 참고할 수 있습니다. 자세한 문법은 8장을 보세요. ㉝ 쇼핑몰·㊶ 호텔에는 새 슬라이드 블록도 함께 들어 있습니다.

이름 뒤에 (2단_좌우)가 붙은 46~60번 템플릿 15종은 하위 메뉴가 작은 드롭다운 대신 화면 좌우로 꽉 차는 풀폭 패널(메가 메뉴)로 펼쳐지는 견본입니다 — 대기업·브랜드 홈에서 흔한 방식으로, 각 하위 항목이 굵은 제목 + 회색 설명으로 나란히 배치됩니다. 문법은 대괄호: 제품[포스:올인원 POS/키오스크:셀프 주문] (8장 참고). 48 전자제품·53 식품·60 방송/엔터에는 슬라이드 블록도 함께 들어 있습니다.

04페이지 — 추가·이동·링크 연결

도구막대 맨 앞 [페이지] 섹션에서 콤보로 전환하고 ＋ 페이지 ✎ 이름 ⧉ 복제 ✕ 삭제로 관리합니다. 메뉴바·푸터는 모든 페이지에 공통으로 표시됩니다.

페이지 순서와 하위(Tree) 구조 010

  • ◀ 순서 / 순서 ▶ — 현재 페이지를 목록에서 앞/뒤로 옮깁니다. 페이지 콤보의 순서, 페이지별 내보내기의 파일 번호(01_, 02_…), 화면정의서의 페이지 순서에 모두 반영됩니다.
  • ⇥ 하위 — 현재 페이지를 하위 페이지로 지정/해제합니다. 페이지 콤보에 └ 들여쓰기로 표시되고 화면정의서에도 반영되어, 페이지가 많을 때 상위-하위 Tree로 정리할 수 있습니다.
  • 하위 지정은 정리·문서화용 표시입니다. 링크 연결은 여전히 "이름 맞추기" 규칙을 따릅니다 — 상위 메뉴 밑으로 묶어 이동시키려면 아래 8장의 드롭다운 메뉴를 함께 사용하세요.

링크는 "이름 맞추기"로 연결됩니다

  • 메뉴바 항목 · 사이드메뉴 항목 · 버튼의 이름이 페이지명과 같으면 자동 연결됩니다. 편집 모드에서 연결된 메뉴 항목에는 작은 ↗ 표시가 붙습니다.
  • 버튼은 이름=색>페이지명 형식으로 직접 지정할 수도 있습니다. 예: 홈으로=회색>홈
  • 이동은 미리보기와 내보낸 HTML에서 동작합니다(편집 모드에서는 클릭이 편집으로 쓰이므로 이동하지 않음).

05행(섹션)과 열 — 레이아웃 15종

  • 화면은 행(가로 섹션)을 위에서 아래로 쌓아 만들고, 각 행은 1~6개의 열로 나뉩니다. 도구막대의 열 분할 콤보에서 고르고 ＋ 행 추가를 누릅니다.
  • 행에 마우스를 올리면 좌상단에 행 도구가 나타납니다 — 배경(밴드) 색 · 레이아웃 변경 · ▲▼ 이동 · ⧉ 복제 · ✕ 삭제.
  • 행 배경을 연회색/연파랑/연노랑/연초록으로 지정하면 포털·제품 홈에서 흔한 색 밴드 섹션이 됩니다.
  • 레이아웃을 줄이면(예: 3열→2열) 넘치는 열의 블록은 마지막 열로 합쳐지며 사라지지 않습니다.

06-1새 블록 3종 자세히 — 슬라이드 · 팝업 · HTML 010

슬라이드(캐러셀)

  • 여러 장의 배너를 한 자리에서 넘겨 보여주는 블록입니다. 그룹 홈페이지에서 추가합니다.
  • 편집: ‹ › 화살표나 아래 점을 눌러 원하는 슬라이드로 이동한 뒤, 제목·설명을 화면에서 직접 입력합니다. 우상단 도구에서 ＋슬라이드/－슬라이드, 배경 톤(어두운/파랑/밝은), 자동넘김/수동, 현재 슬라이드의 배경 이미지 URL을 지정합니다.
  • 미리보기·내보낸 HTML에서는 화살표·점 클릭이 실제로 동작하고, 자동넘김이면 내보낸 HTML에서 5초마다 다음 장으로 넘어갑니다(마우스를 올린 동안은 정지).

팝업(모달)

  • 공지·이벤트 팝업을 표현하는 블록입니다. 그룹 화면 요소에서 추가합니다.
  • 편집 화면에서는 보라 점선 카드로 표시되어 제목·내용을 직접 입력합니다. 속성에서 표시 위치(중앙 모달 / 우하단 카드)와 닫기 버튼 문구를 지정합니다.
  • 미리보기·내보낸 HTML에서는 팝업 블록이 놓인 페이지에 들어올 때 떠 있는 창으로 표시되고, ✕나 닫기 버튼으로 닫힙니다. "오늘 하루 보지 않기" 문구는 설계 표시용(모양만)입니다.
  • 팝업 블록은 페이지당 1개를 권장합니다. 여러 개 두면 겹쳐서 표시됩니다.

HTML 코드

  • 임의의 HTML을 그대로 렌더링하는 블록입니다. 그룹 기본에서 추가한 뒤, 우상단 도구의 〈/〉 코드 버튼으로 코드 편집창을 엽니다.
  • 유튜브: 영상의 [공유 → 퍼가기]에서 복사한 <iframe …> 코드를 붙여넣으면 실제 재생되는 영상이 들어갑니다. 지도: 구글지도·네이버지도의 embed(퍼가기) 코드도 같은 방식입니다.
  • 지도·동영상 "자리" 블록이 모양만 잡는 것과 달리, HTML 블록은 실제로 동작하는 embed를 넣는 용도입니다.
Note
안전을 위해 편집 화면에서는 script가 실행되지 않고, 코드 영역 클릭은 블록 선택으로 쓰입니다(iframe 조작 불가). 내보낸 HTML에서는 붙여넣은 코드가 그대로 실행되므로, 신뢰할 수 있는 코드만 사용하세요.

07데이터 블록 — 표·요약카드·그래프

표 → 그래프 연동

  • 먼저 표 블록을 만들고 데이터를 입력합니다. 첫 컬럼이 라벨(예: 월), 숫자가 많은 컬럼이 값 시리즈로 자동 인식됩니다.
  • 그래프 블록을 추가하고 속성에서 원본 표와 유형(막대·가로막대·누적막대·꺾은선·영역·원형·도넛)·높이를 선택합니다.
  • 표 값을 고치면 그래프가 즉시 다시 그려집니다. 컬럼명을 바꾸면 범례에도 반영됩니다.
  • 내보낸 HTML에는 그래프가 정적 SVG로 담겨 오프라인에서도 그대로 보입니다.
Note
㉙ 관리자 대시보드 템플릿이 "요약카드 + 표→그래프 2종" 연동의 견본입니다.

08메뉴바(GNB)와 푸터

  • 도구막대의 메뉴바/푸터 체크박스로 켜고 끕니다. 둘 다 모든 페이지 공통입니다.
  • 메뉴바에 마우스를 올리면 사이트명 · 메뉴 항목(쉼표) · 톤(어두운/파랑/밝은)을 편집하는 미니 도구가 나타납니다.
  • 푸터의 두 줄은 화면에서 직접 클릭해 수정합니다.

하위(드롭다운) 메뉴 010

  • 메뉴 입력칸에서 괄호와 /로 하위 메뉴를 지정합니다 — 예: 홈, 회사(소개/문의), 고객센터 → "회사"에 마우스를 올리면 소개·문의 드롭다운이 펼쳐집니다.
  • 하위 항목도 상위와 똑같이 이름이 페이지명과 같으면 자동 연결됩니다(편집 모드에서 ↗ 표시). 상위 항목은 자체 페이지 없이 묶음 제목으로만 써도 됩니다.
  • 드롭다운은 편집 캔버스·미리보기·내보낸 HTML(한 파일/페이지별) 모두에서 hover로 동작합니다. ㉘ 인트라넷 포털 템플릿에 예시("회사" 메뉴)가 있고, ㉛~㊺ (2단) 템플릿 15종이 드롭다운 구조의 완성 견본입니다(3장 참고).

좌우 풀폭(메가) 메뉴 010

  • 하위 메뉴를 작은 드롭다운 대신 화면 좌우로 꽉 차는 풀폭 패널로 펼치고 싶으면 괄호 대신 대괄호 [ ]를 씁니다 — 예: 제품·솔루션[링크포스:올인원 POS/키오스크:셀프 주문·결제/포스 하드웨어:단말기·주변기기]
  • 각 하위 항목은 이름:설명 형식으로, 패널에 굵은 제목 + 회색 설명으로 나란히 배치됩니다. 설명(:설명)은 생략할 수 있습니다.
  • 하위 항목의 이름이 페이지명과 같으면 자동 연결됩니다(설명은 표시용). 상위에 마우스를 올리면 메뉴바 바로 아래에 흰색 풀폭 패널이 펼쳐지며, 편집·미리보기·양쪽 내보내기 모두에서 동작합니다.
  • 한 메뉴바에서 일반 드롭다운(괄호)과 풀폭 메뉴(대괄호)를 섞어 쓸 수 있습니다. 항목이 2~3개면 드롭다운, 항목마다 설명을 붙여 소개하고 싶으면 풀폭이 어울립니다.
  • 46~60번 (2단_좌우) 템플릿 15종이 풀폭 메뉴의 완성 견본입니다(3장 참고).

09주석(📌)과 텍스트 모드(📝) — 설계 의도 남기기

주석 모드 (📌) — 번호 핀 + 설명 목록

  • 📌 주석 모드를 켜고 화면 아무 곳이나 클릭하면 빨간 번호 핀이 찍히고, 캔버스 아래 목록에서 설명을 입력합니다.
  • 핀을 다시 클릭하면 삭제됩니다. 주석은 페이지별로 따로 저장됩니다.
  • "이 버튼을 누르면 어떤 API를 호출" 같은 지점별 동작 스펙을 남기는 용도입니다.

텍스트 모드 (📝) — 화면 위에 문구 직접 표시

  • 📝 텍스트 모드를 켜고 화면을 클릭하면 그 자리에 문구를 직접 표시합니다. 편집 창에서 글자 색 5종(빨강·파랑·초록·회색·검정) · 크기(12~20px) · 형광펜 배경을 지정할 수 있습니다.
  • 문구를 드래그해 위치를 옮기고, 클릭하면 다시 편집합니다. 내용을 비우고 저장하면 자동 삭제됩니다.
  • "※ 마감 후에는 수정 불가" 같이 화면에 상시 보여야 할 규칙·주의 문구를 위치 그대로 전달하는 용도입니다. 주석과 마찬가지로 페이지별로 저장됩니다.
  • 주석 모드와 텍스트 모드는 동시에 켤 수 없습니다 — 한쪽을 켜면 다른 쪽이 자동으로 꺼집니다.
Note
핀·설명 목록·텍스트 문구는 모두 내보낸 HTML(한 파일·페이지별)과 화면정의서에 포함됩니다. 화면정의서에는 "📌 주석"과 "📝 화면 문구"가 각각 표로 정리됩니다.

10저장·불러오기·실행취소

  • 💾 저장 — 전체 작업(페이지·블록·메뉴바·푸터·주석·텍스트 문구)을 page_builder_날짜.json으로 내려받습니다.
  • 📂 불러오기 — 저장한 JSON을 다시 엽니다. 이전 버전(001~009)에서 저장한 파일도 자동 변환되어 열립니다.
  • ↶ 실행취소 / ↷ 다시실행 — 최대 60단계. 단축키 Ctrl+Z / Ctrl+Y. 템플릿 적용·페이지 삭제도 되돌릴 수 있습니다.
  • 🗑 초기화 — 모든 페이지를 지우고 빈 화면으로 시작합니다.
Tip
도구 파일 자체에는 작업이 남지 않습니다. 작업을 이어가려면 반드시 JSON 저장을 해 두세요.

11HTML 내보내기 — 한 파일 / 페이지별

두 방식 모두 편집 흔적(점선·버튼·입력 표시) 없이 완성된 화면만 담기고, 주석 핀·설명 목록·텍스트 문구도 포함됩니다. 좁은 화면(모바일)에서는 열이 자동으로 세로로 쌓입니다.

Note — 내보낸 HTML에서 실제로 동작하는 것 010
페이지 이동(메뉴·버튼·사이드메뉴)에 더해 드롭다운 메뉴 펼침, 슬라이드 넘김·자동재생, 팝업 표시·닫기, HTML 블록의 embed(유튜브·지도 등)가 내보낸 파일에서 그대로 동작합니다.

🌐 HTML (한 파일)

  • 모든 페이지가 파일 하나에 담깁니다. 메뉴·버튼·사이드메뉴를 클릭하면 파일 안에서 페이지가 전환됩니다.
  • 파일 한 개만 보내면 되므로 메신저·메일 공유에 편리합니다.

🗂 HTML (페이지별)

  • 페이지마다 파일이 따로 내려갑니다. 파일명은 순번+페이지명 — 예: 01_홈.html, 02_신청.html (폴더에서 페이지 순서대로 정렬됨).
  • 메뉴·버튼·사이드메뉴의 페이지 연결이 파일 간 실제 링크로 변환되어, 01_홈.html에서 "신청"을 누르면 02_신청.html로 이동합니다. 각 파일의 메뉴바에는 현재 페이지에 밑줄이 표시됩니다.
  • 여러 파일이 순차로 내려가므로 브라우저가 "여러 파일 다운로드 허용"을 물으면 허용해 주세요.
  • 파일들을 같은 폴더에 두어야 페이지 간 이동이 동작합니다. 개발자에게 전달할 때는 폴더째 압축해서 보내세요.
Tip
페이지별 개발 담당자가 다르거나 화면 단위로 리뷰·이슈를 관리한다면 페이지별이, 빠른 전체 공유에는 한 파일이 편합니다.

12화면정의서 출력

📑 화면정의서를 누르면 spec_날짜.html 문서가 내려받아집니다. 구성:

  • 0. 공통 영역 — 메뉴바 구성(항목별 페이지 연결 여부), 푸터 문구.
  • 페이지별 — 행 구조(열 분할 비율), 블록 목록 표(No · 위치(행-열) · 유형 · 내용 요약), 📌 주석 표, 📝 화면 문구 표.
  • 블록 요약은 유형에 맞게 정리됩니다. 예: 표는 "컬럼명 — N행", 그래프는 "꺾은선 — 원본: 표(월·매출)", 버튼은 링크 대상까지.

브라우저 인쇄(Ctrl+P)로 PDF 저장하면 그대로 배포 문서가 됩니다.

13자주 묻는 질문(FAQ)

Q. 메뉴를 눌러도 페이지가 안 바뀌어요.

편집 모드에서는 클릭이 편집으로 쓰입니다. 미리보기로 전환하거나 내보낸 HTML에서 확인하세요. 그래도 안 되면 메뉴 항목명과 페이지명이 정확히 같은지(공백 포함) 확인하세요 — 연결된 항목에는 편집 모드에서 ↗ 표시가 붙습니다.

Q. 그래프가 "표를 선택하세요"라고만 나와요.

그래프는 같은 페이지의 표 블록을 원본으로 사용합니다. 표를 먼저 추가하고, 그래프 블록을 클릭 → 속성의 원본 콤보에서 표를 선택하세요. 표에 숫자 컬럼이 하나도 없으면 그릴 수 없습니다.

Q. 입력한 글이 사라졌어요.

텍스트는 입력 후 바깥을 클릭해야 저장됩니다. 저장 전에 페이지를 전환하거나 새로고침하면 유실될 수 있습니다. 습관적으로 💾 저장을 눌러 JSON을 보관하세요.

Q. 템플릿을 골랐는데 원래 작업이 지워졌어요.

Ctrl+Z(실행취소)로 되돌릴 수 있습니다. 템플릿 적용 전 확인창이 뜨므로, 작업 중에는 먼저 JSON 저장을 권장합니다.

Q. 페이지를 추가했는데 상단 메뉴바에 안 보여요.

메뉴바 항목은 자동으로 생기지 않습니다. 편집 모드에서 메뉴바에 마우스를 올리면 우측 상단에 나타나는 메뉴 입력칸에 새 페이지 이름을 쉼표로 추가하세요. 페이지명과 정확히 같으면(공백 포함) 자동 연결되어 ↗ 표시가 붙습니다.

Q. 페이지별 내보내기를 받았는데 파일에서 메뉴를 눌러도 이동이 안 돼요.

페이지별 파일들은 같은 폴더 안에 함께 있어야 서로 이동합니다. 파일 하나만 옮겼거나 이름을 바꾸면 링크가 끊깁니다. 또한 브라우저가 "여러 파일 다운로드"를 차단해 일부 파일만 받아졌을 수 있으니, 내보내기 시 안내된 파일 개수와 실제 받은 개수를 확인하세요.

Q. 슬라이드가 편집 화면에서는 자동으로 안 넘어가요.

자동넘김은 내보낸 HTML에서만 동작합니다(편집·미리보기 중 화면이 저절로 바뀌면 편집을 방해하기 때문). 편집·미리보기에서는 ‹ › 화살표와 점으로 직접 넘겨 확인하세요.

Q. 팝업이 편집 화면에서는 카드로만 보여요.

의도된 동작입니다. 편집 중에는 내용이 가려지지 않도록 점선 카드로 표시되고, 미리보기나 내보낸 HTML에서 페이지에 들어올 때 실제 팝업으로 뜹니다.

Q. HTML 블록에 붙여넣은 유튜브가 편집 화면에서 클릭이 안 돼요.

편집 중에는 embed 영역 클릭이 블록 선택으로 쓰입니다(그래야 블록을 옮기고 지울 수 있습니다). 재생·조작은 미리보기나 내보낸 HTML에서 확인하세요.

Q. 실제 서비스로 그대로 써도 되나요?

내보낸 HTML은 정적 화면입니다. 검색창·입력 폼·로그인 박스는 모양만 있는 설계용이라 실제 동작하지 않습니다. 안내형 페이지·랜딩·사내 공지처럼 "보여주는 화면"은 그대로 사용 가능하고, 기능이 필요한 화면은 이 HTML을 개발 시안으로 전달하면 됩니다. 이때 📌 주석으로 동작 스펙을, 📝 텍스트 모드로 화면 규칙 문구를 함께 담아 보내면 별도 기획서 없이도 전달력이 높아집니다.
